## 운영진용: 링크 공유 방법

### 1. 대회 등록 후 링크 생성

단체 대회 등록이 완료되면 배번 입력 링크를 생성할 수 있습니다.

**링크 형식:**
```
https://dmc-attendance.web.app/my-bib.html?eventId={eventId}
```

**예시:**
```
https://dmc-attendance.web.app/my-bib.html?eventId=evt_2026-04-19_24
```

### 2. 카카오톡 메시지 템플릿

참가자들에게 아래 메시지를 복사하여 전송하세요.

```
🎽 [대회명] 배번 입력 안내

안녕하세요, 동마클 운영진입니다.

대회 당일 기록 매칭을 위해 배번을 입력해주세요.
(1분 소요, 대회 1일 전까지 입력 권장)

👉 배번 입력하기: https://dmc-attendance.web.app/my-bib.html?eventId=evt_2026-04-19_24

📝 입력 방법:
1. 링크 클릭
2. 닉네임 입력 (본인 확인)
3. 배번 번호 입력
4. 저장 완료

❓ 배번을 모르는 경우:
대회 당일 수령하는 번호판(배번)을 확인한 후 입력하시면 됩니다.
입력하지 않아도 기록은 저장되지만, 동명이인 매칭 정확도가 높아집니다.

감사합니다!
```

### 3. 입력 현황 확인 방법

**방법 1: group-detail.html에서 확인**
1. `group.html` 접속
2. 해당 대회 선택 → 상세 페이지 이동
3. 참가자 목록에서 배번 칼럼 확인
   - 배번 입력 완료: 숫자 표시
   - 미입력: "미입력" 표시

**방법 2: Firestore에서 직접 확인 (개발자)**
```javascript
// Firebase Console → Firestore Database
// race_events/{eventId}/participants[].bib
```

### 4. 입력 독려 타이밍

| 시점 | 액션 | 메시지 |
|------|------|--------|
| 대회 3일 전 | 첫 공지 | 위 템플릿 전송 |
| 대회 1일 전 | 리마인더 | "아직 배번을 입력하지 않으신 분들은 입력 부탁드립니다" |
| 대회 당일 | 현장 안내 | "배번 수령 후 즉시 입력 가능합니다" |

### 5. 문제 발생 시 대응

| 문제 | 원인 | 해결 방법 |
|------|------|----------|
| 링크가 열리지 않음 | 잘못된 eventId | eventId 확인 후 재전송 |
| "참가자 아님" 오류 | 닉네임 불일치 | 정확한 닉네임 안내 (대소문자, 공백 주의) |
| 배번이 저장 안 됨 | 네트워크 오류 | 재시도 안내 |
| 잘못된 배번 입력 | 회원 실수 | group-detail.html에서 운영진이 직접 수정 가능 |

## 회원용: 배번 입력 방법

### 1. 링크 접속

운영진이 공유한 링크를 클릭하면 배번 입력 페이지가 열립니다.

**페이지 예시:**
```
┌─────────────────────────────────────┐
│ 🎽 배번 입력                        │
│ 제24회 경기마라톤대회                │
│ 2026-04-19 (토)                     │
└─────────────────────────────────────┘
```

### 2. 닉네임 입력 (본인 확인)

1. "닉네임 입력" 필드에 본인의 동마클 닉네임 입력
2. **정확한 닉네임을 입력해야 합니다** (대소문자, 공백 주의)
3. [다음] 버튼 클릭

**예시:**
```
닉네임 입력: 라우펜더만
```

**주의:**
- 닉네임이 기억나지 않는 경우: 운영진에게 문의
- "참가자 아님" 오류 발생: 닉네임 확인 후 재입력

### 3. 배번 입력

1. "배번 번호" 필드에 본인의 배번 입력
2. **배번은 대회 당일 받는 번호판의 숫자입니다**
3. [저장] 버튼 클릭

**예시:**
```
배번 번호: 12345
```

**배번 형식:**
- 숫자만: `12345`
- 문자+숫자: `A-123` (대회에 따라 다름)
- 앞뒤 공백은 자동 제거됨

### 4. 완료 확인

저장이 완료되면 아래 메시지가 표시됩니다.

```
✅ 저장 완료
배번이 성공적으로 저장되었습니다.
대회 기록이 자동으로 매칭됩니다.
```

### 5. 입력 수정

**배번을 잘못 입력한 경우:**
1. 같은 링크로 다시 접속
2. 닉네임 재입력
3. 올바른 배번 입력
4. 저장 (이전 배번을 덮어씀)

## FAQ

### Q1. 배번을 모르는데 어떻게 입력하나요?

**A:** 배번은 대회 당일 대회 본부에서 수령하는 번호판입니다. 대회 전에는 모르는 것이 정상입니다.

**입력 타이밍:**
- 대회 1일 전에 배번을 미리 알려주는 경우 → 즉시 입력
- 대회 당일 현장에서 수령 → 수령 후 즉시 입력 가능
- 입력하지 못한 경우 → 운영진이 대회 후 처리

### Q2. 배번을 입력하지 않으면 어떻게 되나요?

**A:** 입력하지 않아도 기록은 저장됩니다.

**차이점:**
- **배번 입력 O**: 대회 기록 자동 매칭 (정확도 높음)
- **배번 입력 X**: 이름으로 매칭 (동명이인 시 수동 처리 필요)

**권장:**
- 동명이인이 있는 경우 (예: 김철수, 이영희 등 흔한 이름): 배번 입력 강력 권장
- 드문 이름: 입력하지 않아도 자동 매칭 가능

### Q3. 여러 번 입력할 수 있나요?

**A:** 네, 같은 링크로 여러 번 접속하여 배번을 수정할 수 있습니다.

**작동 방식:**
- 나중에 입력한 배번이 이전 배번을 덮어씀
- 입력 횟수 제한 없음

### Q4. 다른 사람의 배번을 입력할 수 있나요?

**A:** 기술적으로는 가능하지만, **본인의 배번만 입력해주세요**.

**이유:**
- 잘못된 배번 입력 시 기록 매칭 오류 발생
- 운영진이 수동으로 수정해야 함
- 커뮤니티 신뢰 기반 시스템

### Q5. 모바일에서도 가능한가요?

**A:** 네, 모바일 브라우저에서 최적화되어 있습니다.

**지원 환경:**
- iOS Safari
- Android Chrome
- 카카오톡 인앱 브라우저

### Q6. 링크가 열리지 않아요.

**A:** 아래 사항을 확인해주세요.

**체크리스트:**
1. 인터넷 연결 확인
2. 링크를 완전히 복사했는지 확인 (잘린 링크 X)
3. 브라우저에서 직접 열어보기 (카카오톡 인앱 브라우저 오류 시)
4. 운영진에게 링크 재요청

### Q7. "참가자 아님" 오류가 나와요.

**A:** 닉네임이 정확하지 않은 경우 발생합니다.

**해결 방법:**
1. 닉네임 철자 확인 (대소문자, 공백 주의)
2. 운영진에게 본인의 등록 닉네임 문의
3. 본인이 해당 대회에 등록되어 있는지 확인

**예시:**
- 등록 닉네임: `라우펜더만` (공백 없음)
- 잘못된 입력: `라우펜 더만` (공백 있음) → 오류
- 올바른 입력: `라우펜더만` → 성공

### Q8. 배번을 입력했는데 확인하고 싶어요.

**A:** 운영진에게 문의하여 확인 가능합니다.

**확인 방법:**
- 운영진이 group-detail.html에서 참가자 목록 확인
- 또는 같은 링크로 재접속하여 "현재 배번" 표시 확인 (구현 시)

## 성공 지표 추적

### 1. 측정 지표

운영진은 아래 지표를 추적하여 기능 효과를 평가합니다.

| 지표 | 목표 | 측정 방법 | 주기 |
|------|------|-----------|------|
| **배번 입력률** | 30% 이상 | (배번 입력 회원 수 / 전체 참가자 수) × 100 | 대회 직후 |
| **운영진 시간 절감** | 10분 이상 | 대회 당일 매칭 소요 시간 측정 | 대회 당일 |
| **동명이인 오류** | 0건 | 잘못 매칭된 기록 건수 | 대회 직후 |
| **회원 피드백** | 긍정 50% 이상 | 설문 또는 카카오톡 피드백 | 1주일 후 |

### 2. 데이터 수집 방법

**입력률 계산:**
```javascript
// Firestore에서 확인
const participants = raceEvent.participants;
const totalCount = participants.length;
const inputCount = participants.filter(p => p.bib).length;
const inputRate = (inputCount / totalCount) * 100;

console.log(`배번 입력률: ${inputRate.toFixed(1)}%`);
```

**시간 절감 측정:**
- 대회 당일 매칭 시작 시각 기록
- 매칭 완료 시각 기록
- 소요 시간 계산 (이전 대회와 비교)

**오류 건수:**
- group-detail.html에서 수동 수정 횟수 카운트
- 회원 신고 건수 집계

### 3. Phase 2 진행 판단 기준

아래 조건을 **모두 만족**하면 Phase 2 (사진 업로드 + OCR) 진행을 검토합니다.

| 조건 | 기준 | 확인 방법 |
|------|------|----------|
| **입력률** | 30% 이상 | 위 계산식 |
| **회원 피드백** | "숫자 입력 불편" 의견 다수 | 설문 또는 카카오톡 |
| **운영진 수용성** | OCR 검수 작업 수용 가능 | 운영진 회의 |

**Phase 2 미진행 조건:**
- 입력률 30% 미만 → 링크 공유 방식 개선 우선
- "편하다" 피드백 우세 → 현재 방식 유지
- 운영진 검수 부담 우려 → 보류

### 4. 성공 사례 공유

**목표 달성 시:**
1. 운영진 회의에서 결과 공유
2. 회원들에게 감사 메시지 전달
3. 다음 대회에서도 적용

**예시 공유 메시지:**
```

덕분에 대회 당일 매칭 시간이 15분 단축되었고,

```
