## 2. 운영진 내부 공지

### 2.1 운영진 브리핑 (대회 전)

```
[운영진 전용] 배번 입력 기능 런칭 안내

1. 기능 개요:
   - 회원이 본인의 배번을 직접 입력하는 셀프 서비스
   - 운영진은 링크만 공유하면 됨
   - 대회 당일 매칭 시간 단축 기대

2. 운영진 액션:
   ✅ [3일 전] 카카오톡 공지 (위 템플릿 사용)
   ✅ [1일 전] 리마인더 발송
   ✅ [당일] 현장 안내
   ✅ [당일] group-detail.html에서 입력 현황 확인

3. 링크 생성 방법:
   - 형식: https://dmc-attendance.web.app/my-bib.html?eventId={eventId}
   - eventId는 group.html에서 확인 가능

4. 문제 발생 시:
   - 회원 문의: FAQ 참고하여 안내
   - 기술 오류: [개발자] 연락

5. 주의사항:
   - 배번 입력은 선택사항 (강제 X)
   - 입력하지 않아도 기록은 저장됨
   - 운영진이 group-detail.html에서 직접 수정 가능
```

### 2.2 대회 당일 체크리스트

```
[대회 당일] 운영진 체크리스트

□ 대회 시작 전:
  □ 카카오톡 공지 발송 (배번 입력 안내)
  □ group-detail.html 접속하여 입력 현황 확인

□ 대회 진행 중:
  □ 회원 문의 대응 (FAQ 참조)
  □ 입력 오류 발생 시 직접 수정

□ 대회 종료 후:
  □ 최종 입력 현황 확인
  □ [기록 매칭] 버튼으로 자동 매칭 시작
  □ 배번 미입력 회원 수동 매칭
  □ 매칭 소요 시간 기록 (성공 지표)

□ 1주일 후:
  □ 입력률 계산
  □ 회원 피드백 수집
  □ 운영진 회의에서 결과 공유
```

```

```

## 4. Phase 2 진행 판단 기준

### 4.1 Phase 2 개요

**Phase 2: 사진 업로드 + OCR**
- 회원이 배번 사진을 업로드
- Vision API로 자동 숫자 인식
- 운영진이 검수 후 확정

**장점:**
- 입력 편의성 향상 (사진만 찍으면 됨)
- 숫자 입력 오류 감소

**단점:**
- 운영진 검수 작업 추가
- 구현 복잡도 증가 (1-2일 소요)

### 4.2 진행 조건 (AND)

| 조건 | 기준 | 확인 |
|------|------|------|
| **입력률** | 30% 이상 | ☐ |
| **회원 피드백** | "숫자 입력 불편" 의견 다수 (30% 이상) | ☐ |
| **운영진 수용성** | OCR 검수 작업 수용 가능 | ☐ |

**예상 검수 작업량:**
- 참가자 85명 × 입력률 30% = 25건
- 검수 시간: 건당 10초
- 총 소요 시간: 약 5분

### 4.3 의사결정 프로세스

```
1주일 후 운영진 회의:

1. 성과 보고서 공유
2. 회원 피드백 분석
3. Phase 2 진행 여부 논의
4. 의사결정
   - 진행 → 개발 착수
   - 보류 → 현재 방식 유지
   - 중단 → 기능 개선 재검토
```

## 5. 예상 FAQ (운영진용)

### Q1. 회원이 배번을 잘못 입력했어요.

**A:** group-detail.html에서 직접 수정 가능합니다.

**수정 방법:**
1. group.html → 해당 대회 선택
2. 참가자 목록에서 해당 회원 찾기
3. 배번 칼럼 클릭 → 수정 → 저장

### Q2. 회원이 "링크가 안 열린다"고 해요.

**A:** 아래 체크리스트로 확인하세요.

**체크리스트:**
1. 링크가 완전한가? (복사 시 잘림 확인)
2. eventId가 정확한가?
3. 인터넷 연결이 되어 있는가?
4. 브라우저에서 직접 열어보기

**해결 안 되면:**
- 링크 재전송
- 또는 운영진이 직접 입력

### Q3. 입력률이 낮아요 (10% 미만).

**A:** 아래 방법으로 독려하세요.

**독려 방법:**
1. 리마인더 재발송 (대회 1일 전)
2. 카카오톡 개별 메시지 (동명이인 우선)
3. 대회 당일 현장 안내 (배번 수령 직후)

**입력률 목표:**
- 최소: 10% (동명이인만이라도)
- 목표: 30%
- 이상: 50% 이상

### Q4. 대회 당일 링크가 작동하지 않아요.

**A:** 즉시 개발자에게 연락하고, 임시 조치로 운영진이 직접 입력하세요.

**임시 조치:**
1. group-detail.html 접속
2. 회원이 구두로 배번 알려주면 직접 입력
3. 대회 후 원인 파악

### Q5. 회원이 "본인이 아닌 다른 사람의 배번을 입력했다"고 해요.

**A:** group-detail.html에서 수정 가능합니다.

**대응:**
1. 해당 회원의 배번을 정확한 값으로 수정
2. 카카오톡으로 "본인 배번만 입력" 안내 재공지
3. 악의적 사용 의심 시 운영진 회의

## 6. 롤백 계획

### 6.1 문제 발생 시나리오

| 문제 | 심각도 | 롤백 필요 |
|------|--------|----------|
| 링크가 열리지 않음 | 높음 | 검토 |
| 저장이 안 됨 | 높음 | 검토 |
| 입력률 5% 미만 | 중간 | 불필요 |
| 회원 부정적 피드백 다수 | 중간 | 검토 |

### 6.2 롤백 절차

**1. 긴급 롤백 (링크 작동 불가 시):**
```
1. my-bib.html 비활성화 (hosting에서 제거)
2. 카카오톡 공지: "기술 오류로 배번 입력 서비스 일시 중단"
3. 운영진이 group-detail.html에서 직접 입력
4. 개발자가 원인 파악 및 수정
```

**2. 점진적 철수 (입력률 저조 시):**
```
1. 다음 대회부터 링크 공유 중단
2. 기능은 유지 (group-detail.html API 사용 중)
3. 운영진 회의에서 재검토
```
