## 1. 유용성 (Usefulness) — 9/10

### ✅ 강점
- **핵심 작업 지원**: 일괄 저장 조건(85명 전원 선택)이 명확히 구현됨
- **진행 상태 추적**: 프로그레스 바로 진행률 실시간 확인 가능
- **미처리 식별**: 누락된 작업을 목록으로 제시

### ⚠️ 약점
- **일괄 저장 이후 flow**: 저장 후 다시 돌아왔을 때 무엇을 볼 수 있는가? (통계? 참가자 목록?)
  - **개선**: 저장 완료 시 "확정됨" 상태로 UI 전환 (통계 표시)

**점수**: 9/10 (flow 완결성 -1)

## 2. 사용성 (Usability) — 9/10

### ✅ 강점
- **Sticky footer**: 일괄 저장 버튼이 항상 접근 가능
- **필터 칩**: "미처리만" 필터로 빠른 작업 가능
- **섹션 접기**: 자동 매칭 78명을 접어서 집중도 향상
- **클릭 연동**: 미처리 목록 클릭 → 스크롤 + 하이라이트

### ⚠️ 약점
- **동명이인 UI**: 라디오 버튼 2개가 카드형으로 분리되어 있어 모바일에서 스크롤 필요할 수 있음
  - **현재**: 세로 배치 (공간 많이 차지)
  - **개선 가능**: 가로 배치 또는 compact 모드

**점수**: 9/10 (모바일 동명이인 UX -1)

## 3. 명확성 (Clarity) — 10/10

### ✅ 강점
- **용어 일관성**: "선택 완료" (메모리) vs "일괄 저장" (Firestore) 명확
- **진행 상태 메시지**: "⚠️ 미처리 5명 — 모든 참가자의 기록을 선택해야 일괄 저장할 수 있습니다"
- **시각적 위계**:
  - 자동 매칭 78명: opacity 0.6 (덜 중요)
  - 수동 처리 7명: opacity 1.0 (집중 필요)
- **색상 코딩**:
  - 초록(✅): 선택됨
  - 노랑(⚠️): 동명이인
  - 빨강(🔴): 기록 없음

## 4. 오류 처리 (Error Handling) — 9/10

### ✅ 강점
- **실패 모달**: 오류 상세 + 재시도 버튼
- **재시도 안전성**: "중복 저장 안 됨" 명시
- **조건 검증**: 미처리 있으면 일괄 저장 차단

### ⚠️ 약점
- **네트워크 오류 구분 부재**:
  - 현재: 모든 오류를 동일하게 처리
  - **개선**: 네트워크 오류(재시도 가능) vs 데이터 오류(수정 필요) 구분
  - 예: "네트워크 오류입니다. 인터넷 연결을 확인하세요."

**점수**: 9/10 (오류 유형 구분 -1)
