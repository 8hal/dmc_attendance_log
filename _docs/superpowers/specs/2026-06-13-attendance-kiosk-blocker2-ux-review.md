## 1. 점검 범위

| 영역 | 이번 범위 |
|------|-----------|
| 키오스크 `mode=kiosk` | 명부 외 출석 경로, CTA 위치, 모달 문구, 개인 링크 숨김 |
| 개인 v2 (`attendance-v2.html`) | 용어 스윕, 보조 플로우 (현장은 키오스크 권장) |
| 에러 복구·로깅 | UX에 영향 주는 메시지·완료 화면만 (기술 상세는 설계 §4) |

---

## 2. 기획 원칙 (확정)

1. **출석 명부 ≠ 정회원 명단** — 화면에는 「출석 명부」만 사용.
2. **명부 외 UX 1종** — 정회원/준회원/신규 UI 분기 없음 (운영 명부에서만 구분).
3. **사용자-facing에 「게스트」 미사용** — API·DB 필드명은 레거시 유지.

---

## 3. 키오스크 화면 흐름 (v5)

```
[홈]
  ├─ 닉네임으로 찾기 → [첫 글자] → [닉네임 목록 + 하단 CTA]
  ├─ 팀으로 찾기     → [팀 선택]  → [닉네임 목록 + 하단 CTA]
  └─ 당일 출석 현황 보기 (모달)

[닉네임 목록]  ← 명부 외 CTA는 여기만
  · 그리드: 출석 명부 닉네임 카드 (출석/완료)
  · 목록 최하단(고정): 「명단에 없어요 — 여기서 출석」 → 모달 → 출석 완료

[홈에는 명부 외 CTA 없음]
```

### 3.1 홈 (`kioskHomePanel`)

| 요소 | 문구·동작 |
|------|-----------|
| 타일 1 | 닉네임으로 찾기 / 내 이름 첫 글자 |
| 타일 2 | 팀으로 찾기 |
| 보조 버튼 | **당일 출석 현황 보기** (`status` 목록) |
| 개인 링크 | **완전 숨김** (`kioskPersonalLink` — 자봉용 링크도 없음) |
| 명부 외 CTA | **없음** (v4 홈 CTA 폐기) |

### 3.2 닉네임 목록 (`kioskMemberPanel`)

**진입:** 첫 글자 선택 후 **또는** 팀 선택 후.

| 영역 | 내용 |
|------|------|
| 헤더 | ← 뒤로, 제목(첫 글자 또는 팀명) |
| 그리드 | 출석 명부 닉네임 카드. 스크롤 영역 |
| **목록 하단 (고정)** | CTA 버튼 — 그리드와 분리, 항상 표시 |

**CTA (확정안):**

```
[ 명단에 없어요 — 여기서 출석 ]
```

- 터치 영역 ≥ 44px (`kiosk-secondary-button` 스타일)
- 목록에 다른 사람 이름이 있어도 **항상** 하단에 표시 (자신의 닉이 없을 때 찾는 맥락)
- 그리드가 **비어 있을 때**도 동일 CTA (empty 문구 + 하단 CTA)

**empty 문구 (그리드만 비었을 때):**

「출석 명부에 해당 닉네임이 없습니다」

### 3.3 모달 (`#guestModal` — 키오스크 진입)

| 항목 | 문구 |
|------|------|
| 제목 | 출석 명부에 없는 경우 출석 |
| 닉네임 | 필수 입력 |
| 정모·날짜 | `kioskState` 고정, 입력 필드 숨김 (URL·상단 타이틀로만 표시) |
| 안내 1 | 출석 명부에 없는 경우입니다. 출석은 기록되며, 명부 반영은 운영진이 따로 합니다. |
| 안내 2 | 명부 수정이 필요하면 **게살볶음밥**에게 알려주세요. |
| 제출 | 출석 등록 |
| 취소 | 취소 |

### 3.4 완료

| 항목 | 동작 |
|------|------|
| 메시지 | 「{닉네임}님 · 출석 완료」 |
| 월 통계 카드 | 숨김 (`memberId` 없음) |
| 복귀 | 3초 후 홈 |

### 3.5 실패 (명부 내 출석과 동일)

POST 실패 → 로그 → roster 재로드 → 이미 출석이면 「이미 출석 완료」, 그 외 IT 안내.

## 4. 개인 모드 v2 (보조)

| 위치 | 문구 |
|------|------|
| 검색 도움말 | 출석 명부에서 닉네임을 검색해 선택하세요. |
| 검색 0건 | 「…에 맞는 닉네임이 없어요…」 + **현장은 키오스크 출석을 이용해 주세요.** |
| 하단 버튼 | 출석 명부에 없는 경우 |
| 모달 | 개인 모드: 날짜·정모 입력 유지. 안내에 게살볶음밥 문구 포함 가능 |

---

## 5. 용어·노출 금지

구현 후 수동 스윕:

- [ ] UI에 「정회원 명단」 없음
- [ ] UI에 「게스트」(버튼·성공 문구·모달 제목) 없음
- [ ] 키오스크에서 「개인 출석 화면」 링크 없음

## 6. v4 → v5 변경 요약

| 항목 | v4 | v5 (본 점검) |
|------|-----|----------------|
| 명부 외 CTA 위치 | 홈 + empty 그리드 | **닉네임 목록 화면 하단만** (홈 제거) |
| CTA 문구 | 출석 명부에 없어요 | **명단에 없어요 — 여기서 출석** |
| 운영 연락 | 운영진 일반 문구 | **게살볶음밥** 담당자명 유지 |
| `kioskPersonalLink` | 숨김 | **완전 숨김** (확정) |
| 당일 현황 버튼 | 현재 출석 명단 보기 | **당일 출석 현황 보기** (유지) |

## 8. 구현 시 레이아웃 메모 (FE)

`kioskMemberPanel` 구조 권장:

```html
<div id="kioskMemberPanel" class="kiosk-panel">
  <div class="kiosk-member-head">…</div>
  <div id="kioskMemberGrid" class="kiosk-member-grid">…</div>
  <button id="kioskMemberNotOnRosterBtn" class="kiosk-secondary-button kiosk-not-on-roster-button">
    명단에 없어요 — 여기서 출석
  </button>
</div>
```

- 홈의 `kioskNotOnRosterBtn` **제거**
- empty 시: 그리드 안 empty 문구 + 하단 버튼은 패널 고정으로 유지
