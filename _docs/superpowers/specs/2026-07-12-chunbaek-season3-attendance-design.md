# PRD: 춘백 시즌3 — 100일 출석 관리 (MVP)

> 작성일: 2026-07-12  
> 상태: 기획 확정  
> 대상: 동마클 「춘천 마라톤 100일 준비방」 춘백 시즌3  
> 관련: DMC attendance log와 **별도** 프로젝트로 구현 권장 (도메인·인프라 분리)

---

## 1. 배경 및 문제

### 1.1 프로그램 개요

동마클 「춘천 마라톤 100일 준비방」의 **춘백 시즌3**는 100일 동안 함께 훈련하며 춘천마라톤·제주마라톤 완주를 목표로 하는 시즌제 프로그램이다.

**시즌 미션 (출정식 기준)**

1. 춘마·제마 DNF 0%
2. 전원 부상 없이 완주
3. PB 달성률 60% 이상
4. 새벽훈련 출석률 주 3회 이상 (토요일 동마클 훈련 인정)

**운영 규칙 (출석 관련)**

- 출석률은 1주 단위로 확인
- 주 3회 미달 시 다음 훈련에서 CU 커피·간식 준비 (패널티)
- 개인 사정은 사전에 관리자에게 알리면 예외 인정
- 4주 연속 미달 시 참여 여부 상의

### 1.2 현재 방식의 한계

| 방식 | 문제 |
|------|------|
| **카카오톡 단톡** | 일정·참석 투표는 충분하나, **주·월·시즌 통계**가 없음. 단톡 사진과 출석 집계가 분리됨 |
| **구글 시트** | 회원이 본인 행을 찾기 어렵고, 모바일 입력·수식 유지·개인 대시보드가 불편함 |
| **기성 앱 (클루보, Strava 등)** | 출석·RSVP는 되나 **100슬롯 커리큘럼 + 시즌 출석 집계**에 맞지 않음. 카카오톡 투표 API 연동 불가 |
| **DMC attendance log** | QR 정모 출석·대회 기록 SSOT에 최적화. **100일 시즌·슬롯 단위 출석**과는 도메인이 다름 |

### 1.3 카카오톡 연동 불가 (조사 결과)

- 단톡 일정·투표 결과를 읽는 **공식 API 없음** (개인정보 보호로 거절 사례 다수)
- 톡캘린더 API는 **개인 캘린더**용이며 단체방 일정과 무관
- PC에서 단톡 메뉴를 외부 웹으로 원격 제어하는 것도 **불가**
- **절충안**: 단톡은 소통·응원·사진 공유 유지, 숫자·출석은 **웹 링크**로 수집

### 1.4 최종 목표 (시즌 전체)

> 모든 참여자가 **원하는 기록을 내도록 돕는 것** — 궁극적으로는 춘마·제마 **공식 기록 (PB/DNF)** 이지만, 그 전제 조건으로 **꾸준한 훈련 출석**이 가장 중요하다.

**본 문서(MVP)는 출석에 한정한다.** 마일리지·몸무게·대회 기록은 Phase 2 이후.

## 2. 목표 및 성공 기준

| 목표 | 성공 기준 |
|------|----------|
| **출석이 코어** | 회원이 매일 **10초 이내**에 오늘 슬롯 출석 완료 |

| **시트 대체** | 구글 시트 없이 주·월·시즌 출석 집계 가능 |

## 3. 핵심 도메인 모델

### 3.1 기본 개념

```
시즌 (100일)
  └─ 슬롯 1 ~ 100   (각각 = 1일)
        ├─ 훈련 내용 (시즌 시작 전 운영진이 고정)
        └─ 출석 기록 (회원 본인 입력)
              ├─ 출석 여부
              ├─ 사진 (선택, 나중에 필수 전환 가능)
              └─ 메모 (선택)
```

### 3.2 출석 상태 정의

출석 관련 개념은 **슬롯 유형**(프로그램)과 **회원별 상태**(개인)로 나눈다.  
「휴식」이라는 말은 개인이 쉬는 것으로 오해될 수 있어, 프로그램 전체 휴무는 **프로그램 휴무일**로 부른다.

#### A. 슬롯 유형 (100슬롯, 운영진 사전 설정)

| 슬롯 유형 | 필드 | 의미 | 회원 행동 | 주간 집계 |
|----------|------|------|----------|----------|
| **훈련일** | `isProgramOff: false` | 그날 프로그램상 훈련이 있는 날 | 출석 표시 가능 | 분모에 포함 |
| **프로그램 휴무일** | `isProgramOff: true` | **전원** 훈련 없음 (커리큘럼상 off) | 아무 것도 안 함 | **제외** (그날 출석 의무 없음) |

> **개인이 그날 안 뛰는 것**은 프로그램 휴무가 아니다. 훈련일인데 참여하지 않으면 **미출석**이며, 사전 승인된 사정은 운영진이 **예외**로 처리한다.

#### B. 회원별 상태 (훈련일 슬롯만)

| 상태 | 의미 | 누가 설정 | 주 3회 집계 |
|------|------|----------|------------|
| **미표시** | 아직 안 함 / 불참 | — | 미달 가능 |
| **출석** | 그날 훈련에 참여 (새벽·동마클 토요일·개인 훈련 포함) | **회원** (`[출석하기]`) | +1 |
| **예외** | 사전 승인된 불참 (출장·건강·경조사 등) | **운영진만** | **제외** (분모·분자 모두) |

**회원 앱에 예외 버튼 없음** — 주 3회 회피를 막기 위함.

#### C. 예외 처리 흐름 (확정)

```
회원: 단톡방에 사전 알림 ("이번 주 수요일 출장으로 불참")
  ↓
운영진: 단톡 확인 후 운영 화면에서 해당 회원·슬롯을 [예외] 처리
  ↓
시스템: 해당 슬롯은 주 3회 계산에서 제외
```

- 예외는 **출석이 아님** (`attended: false`, `exception: true`)
- 운영진은 필요 시 **출석 대리 입력**도 가능 (회원이 앱에서 깜빡한 경우)
- 예외 처리 시 메모 권장 (예: "출장, 단톡 4/10")

#### D. MVP에서 다루지 않는 것

| 개념 | MVP | 이후 |
|------|-----|------|
| **참석 예정** (RSVP) | Out of scope — 단톡 투표 유지 | Phase 1.5 검토 |
| **개인 휴무** | 별도 상태 없음 → 미출석 | — |

#### E. 주 3회 계산

```
해당 주 훈련일 슬롯(isProgramOff === false) 중
  예외 슬롯 제외한 분모 대비
  출석(attended === true) ≥ 3  → 달성
```

프로그램 휴무일 슬롯은 애초에 출석 대상이 아니다.

### 3.3 집계

| 단위 | 계산 |
|------|------|
| **주간** | 해당 주 슬롯 중 출석 횟수, 목표 3회 달성 여부 |
| **월간** | 해당 월 슬롯 중 출석 횟수·출석률 |
| **시즌** | 100슬롯 중 출석한 슬롯 수 (프로그램 휴무일·예외 제외 규칙 적용) |

집계는 `attendance` + `slots` 를 쿼리하여 계산한다. MVP에서 별도 집계 컬렉션은 두지 않는다.

## 4. 범위

### 4.1 MVP (In Scope)

- 100개 `slots` (날짜, 주차, 훈련 라벨, 프로그램 휴무 여부)
- 슬롯별 **출석** (회원 본인 입력)
- 출석 **사진** (선택, `season_config.photoRequired` 로 추후 필수 전환 가능)
- 메모 (선택)
- **주·월·시즌** 출석 집계
- 화면: **오늘 · 내 100일 · 팀 · 나 · 운영**
- 운영진: 출석 수정, 예외 처리, 슬롯 import/편집

### 4.2 Out of Scope (MVP)

| 항목 | 비고 |
|------|------|
| 마일리지 (km) | Phase 2 |
| 몸무게 | Phase 2 |
| 시즌 목표·대회 PB/DNF | Phase 3 |
| RSVP (출석 예정 투표) | 추후 슬롯 필드 확장 |
| 카카오톡 API 연동 | 불가 |
| 회비 5만원 관리 | 시트·단톡 공지로 운영 가능 |
| 푸시 알림 | 단톡 공지로 대체 |
| Strava / Garmin 연동 | Phase 2+ 검토 |

### 4.3 Phase 로드맵

| 단계 | 내용 |
|------|------|
| **MVP** | 100슬롯 + 출석 + 사진(선택) + 주/월/시즌 집계 + 팀/운영 |
| **Phase 2** | 슬롯에 마일리지·몸무게 필드 추가, 개인 그래프 |
| **Phase 3** | 춘마·제마 공식 기록, PB/DNF, 팀 미션 달성률 (DMC `race_results` 패턴 참고) |

## 6. 정보 구조 (IA)

### 6.1 모바일 — 하단 탭 4개

```
[ 오늘 ]  [ 내 100일 ]  [ 팀 ]  [ 나 ]
```

| 탭 | 역할 |
|----|------|
| **오늘** | 오늘 슬롯 출석 (메인 활동, 매일 방문) |
| **내 100일** | 100슬롯 타임라인 + 주/월/시즌 출석 집계 |
| **팀** | 팀 시즌 진행률, 주차별 출석 현황 |
| **나** | 닉네임, 시즌 출석 요약 (MVP 최소) |

### 6.2 운영진

- URL `/admin` (또는 비공개 경로 + 간단 인증)
- **PC 레이아웃** (출석 그리드)

### 6.3 단톡 연동 (비API)

- 단톡 고정 공지에 **「오늘 출석」웹 링크** 1개
- 사진은 단톡에도 올릴 수 있으나, 앱에도 **선택** 업로드 가능 (운영진이 앱만 봐도 확인 가능하도록)

## 7. 화면 설계

### 7.1 오늘 (홈) — 메인 활동

가장 자주 쓰는 화면. 시즌 중 매일 이 화면으로 출석한다.

```
┌─────────────────────────────┐
│  춘백 시즌3    42/100일      │
│  출석 68일 · 시즌 68%        │
├─────────────────────────────┤
│  42일차 · 4월 15일 (화)      │
│  📋 5km 인터벌 + 코어 20분   │  ← slots.trainingLabel
├─────────────────────────────┤
│                             │
│     [  ✓  출석하기  ]       │  ← 메인 CTA
│                             │
│  사진 (선택)  [ + 추가 ]    │
│  메모 (선택)  [        ]    │
├─────────────────────────────┤
│  이번 주  2 / 3회 출석      │
└─────────────────────────────┘
```

**동작**

- **출석하기** 탭 → `attended: true` 저장 → 토스트 (예: "42일차 출석 완료 · 이번 주 3/3")
- 이미 출석 → **출석 완료 ✓** 상태, 수정·사진 추가 가능
- **프로그램 휴무일** → "오늘은 프로그램 휴무일입니다" (출석 버튼 없음)
- 시즌 전/후 → 안내 메시지

**검증 (MVP)**

- `photoRequired: false` (기본) → 사진 없이 제출 가능
- 추후 `photoRequired: true` → 사진 없으면 제출 차단 (설정만 변경)

### 7.2 내 100일

주차별 접이 타임라인. 슬롯 탭 시 출석 폼(오늘 화면과 동일).

```
[ 주간 ] [ 월간 ] [ 시즌 ]   ← 집계 서브탭

▼ 7주차  4/7 ~ 4/13
   ●●○○○  2/3회 출석
   36일  인터벌    ✓  📷
   37일  (휴무)    —
   38일  장거리    ✓
   39일  —         미출석
   ...
▶ 6주차
```

**표기**

- ● 출석 / ○ 미출석 / — 프로그램 휴무 / 🟡 예외
- 오늘 슬롯 강조

**주간 집계 예시**

```
7주차 요약
  출석 2 / 3회 (목표 미달)
  출석률 29% (출석 대상 슬롯 대비)
```

### 7.3 팀

```
시즌 출석 68% (팀 평균)
이번 주 3회 달성  12/18명

7주차
  김○○  ███  3/3  ✓
  이○○  ██░  2/3
  박○○  █░░  1/3
```

- 출석 횟수·주 목표 달성 중심 (km 경쟁·순위 없음)
- 몸무게·사진은 팀에 비공개 (MVP 해당 없음)

### 7.4 나 (MVP 최소)

```
닉네임  홍길동
시즌 출석  68/100일
이번 주    2/3회
```

Phase 2에서 목표·몸무게·km 추가.

### 7.5 운영 (PC)

```
┌──────────┬────────────────────────────────┐
│ 주차 선택 │  7주차  미출석 4명  [CSV↓]      │
│ 슬롯 편집 │ ┌────┬────┬────┬────┐          │
│ 시즌설정  │ │슬롯│김○○│이○○│박○○│ ...     │
│          │ ├────┼────┼────┼────┤          │
│          │ │ 36 │ ✓📷│ ✓  │ —  │          │
│          │ │ 38 │ ✓  │ —  │ ✓  │          │
│          │ └────┴────┴────┴────┘          │
└──────────┴────────────────────────────────┘
```

- 셀 클릭 → `[출석]` `[미출석]` `[예외]` (운영진 전용, 회원 앱에는 예외 없음)
- 📷 → 사진 모달
- **주 3회 미달** 필터 (패널티 후보)
- 슬롯 편집: 100일 훈련표 엑셀/CSV import

## 8. 유저 시나리오

### 8.1 회원 — 당일 출석 (10초)

1. 단톡 링크 또는 북마크로 **오늘** 탭 진입
2. 닉네임 선택 (최초 1회, localStorage 저장)
3. **[출석하기]** 탭
4. (선택) 사진 추가
5. 완료 토스트 · 이번 주 N/3 갱신

### 8.2 회원 — 빠진 날 보완

1. **내 100일** → 해당 주 펼치기
2. 미출석 슬롯 탭 → 출석하기

### 8.3 운영진 — 예외 처리

1. 회원이 **단톡방**에 사전 불참 알림 (출장·건강·경조사 등)
2. 운영진이 단톡 확인
3. **운영** 화면 → 해당 회원·슬롯 → **[예외]** 설정 + 메모
4. 주 3회 집계에서 자동 제외

### 8.4 운영진 — 주간 점검

1. **운영** → 이번 주차 선택
2. 미출석·주 3회 미달 회원 확인
3. 단톡 독촉 (앱 밖)
4. 깜빡한 출석은 운영진이 **[출석]** 대리 입력

### 8.5 운영진 — 시즌 시작 전

1. 100슬롯 훈련표 CSV import
2. `season_config` (시작일, `weeklyTarget: 3`, `photoRequired: false`) 설정
3. 단톡에 출석 링크 고정 공지

## 9. 데이터 모델

### 9.1 Firestore 컬렉션 (제안)

```javascript
// 시즌 전 고정 — 100건
slots {
  slotId: 1..100,           // 또는 문서 ID
  dayIndex: 1..100,
  date: "2026-04-15",       // ISO date
  week: 7,                  // 주차
  trainingLabel: "5km 인터벌 + 코어 20분",
  trainingType: "interval", // optional: interval | long | dawn | dongmak_sat | off
  isProgramOff: false       // true = 프로그램 휴무일 (전원 훈련 없음, 출석 집계 제외)
}

// 회원
members {
  nickname: "홍길동",
  realName: "",             // optional, Phase 3 대회 기록용
  hidden: false
}

// 회원 × 슬롯
attendance {
  memberId: "...",
  slotId: 42,
  attended: true,           // exception이 true이면 false
  exception: false,         // 운영진만 true 설정 (단톡 사전 알림 확인 후)
  exceptionNote: "",        // 예: "출장, 단톡 4/10"
  exceptionSetBy: "",       // 운영진 memberId (optional)
  photoUrl: "",             // optional, Firebase Storage
  note: "",
  updatedAt: Timestamp
}

// 시즌 설정 (단일 문서 또는 컬렉션 1건)
season_config {
  seasonId: "chunbaek-s3",
  title: "춘백 시즌3",
  startDate: "2026-...",
  endDate: "2026-...",
  weeklyTarget: 3,
  photoRequired: false      // true로 변경 시 사진 필수
}
```

### 9.2 Storage

```
attendance/{seasonId}/{slotId}/{memberId}.jpg
```

- 클라이언트 업로드 전 리사이즈 (긴 변 1200px, JPEG ~80%)

### 9.3 집계 쿼리 (개념)

- **주간 출석 수**: `attendance` where `slotId` in (해당 주 slots) and `attended == true`
- **주 목표 달성**: 출석 대상 슬롯(`!isProgramOff`, `!exception`) 대비 `attended === true` ≥ `weeklyTarget`
- **시즌 진행률**: 출석한 slot 수 / 출석 대상 slot 수 (현재 일자까지)
