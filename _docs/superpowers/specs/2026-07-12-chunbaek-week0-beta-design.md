# 춘백 S3 — 0주차 베타 (Week 0) 설계

> **승인:** 2026-07-12 — B(훈련+출석)  
> **갱신:** 2026-07-12 — 베타 기간 **본시즌과 동일 UX·집계** (1주차 전 DB 클리어 전제)  
> **기간:** 2026-07-13(월) ~ 2026-07-19(일)  
> **본시즌 시작:** 2026-07-20(월) 1주차

---

## 목적

베타(7/13~19) 동안 **1주차(7/20~) 훈련표·출석 데이터와 분리**된 테스트 주차를 제공한다.  
참가자·운영진이 **실제 시즌과 동일한 화면·집계**로 피드백을 수집할 수 있게 한다.

## 데이터 모델

| 필드 | 0주차 | 본시즌 |
|------|-------|--------|
| `week` | `0` | `1`~`15` |
| `dayIndex` | `901`~`907` | `1`~`100` |
| `date` | 7/13~7/19 | 7/20~ |
| 컬렉션 | `chunbaek_slots` (동일) | 동일 |

`season_config` 추가 필드:

- `betaWeekStartDate`: `2026-07-13`
- `betaWeekEndDate`: `2026-07-19` (또는 `startDate` 전날 자동)

## 동작

| 영역 | 규칙 |
|------|------|
| admin | `0주차 (베타)` 선택·저장. 베타 기간 기본 주차 = 0 |
| 회원 홈 | 베타 날짜에 `today-slot` 활성. 일차는 **1~7** 표시 (`displayDayIndex`) |
| 출석 | 베타 기간 + `week===0` 슬롯만. 7/20 이후 베타 출석 불가 |
| 시즌 집계 | **베타 기간(`today < startDate`)**: 0주차만 집계 (출석률·일차·팀 weekMet 포함) |
| 시즌 집계 | **본시즌 시작 후**: `week>0`만 집계 (0주차 자동 제외 — DB 클리어 이중 안전) |
| 내 100일 | 시즌 시작 전에만 0주차 표시, `weekLabel: "0주차"` |
| 7/20 이후 | 0주차 타임라인 숨김, 1주차부터 본시즌 |

## 운영

1주차(7/20) 시작 전 `chunbaek_attendance` 베타 데이터 **삭제** 예정.  
삭제하지 않아도 본시즌 집계에는 0주차가 포함되지 않음.

## 시드

```bash
node scripts/seed-chunbaek-week0.js --dry-run
# 승인 후
node scripts/seed-chunbaek-week0.js
```
