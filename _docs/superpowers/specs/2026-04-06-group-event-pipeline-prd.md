# PRD: 단체 대회 파이프라인

> 작성일: 2026-04-06
> 상태: 승인됨
> 우선순위: P1

---

## 1. 배경 및 문제

동탄 마라톤 클럽(DMC)은 연 5~10회 단체 대회에 참가한다. 현재 시스템은 완전히 사후(reactive) 방식이다.

- 운영자가 누가 참가하는지 시스템에서 관리하지 않는다
- 대회 후 기록 수집을 매번 수동으로 트리거해야 한다
- 기록이 누락된 참가자를 자동으로 탐지할 수 없다
- ops.html에 접근 제어가 없어 누가 무엇을 수정했는지 추적 불가능하다

**핵심 고통:** 단체 대회 후 "누구 기록이 없지?"를 운영자가 수동으로 일일이 대조해야 한다.

---

## 2. 목표 및 성공 기준

| 목표 | 성공 기준 |
|---|---|
| 수동 대조 작업 제거 | **첫 단체 대회 적용 후 운영자 수동 대조 작업 0회** |
| 기록 자동 수집 | 대회일 오후 3시, 소스 매핑된 단체 대회 자동 스크랩 |
| 갭 자동 탐지 | 스크랩 후 참가자 vs 기록 비교 결과 자동 표시 |
| 접근 제어 | ops.html은 오너만, group.html·report.html은 운영자만 접근 가능 |

---

## 3. 범위

### In Scope

- `group.html` 신규 페이지: 단체 대회 관리 (참가자 등록·갭 탐지·기록 확정)
- `ops.html` 오너 비밀번호 분리 (`DMC_OWNER_PW`) + 기록 소스 매핑 UI
- `ops.html` → 대회 당일/전날 오너에게 소스 미매핑 강력 알림
- 백엔드 API: 단체 대회 CRUD, 참가자 등록, 갭 탐지
- 기록 확정: 기존 `confirm` API 재사용 (group.html이 새 진입점)
- Cloud Scheduler: 대회일 오후 3시 자동 스크랩 트리거

### Out of Scope

- 개인 회원 자발 참가 등록 (my.html) — 추후 구현
- bib 사전 입력 / 레디샷 OCR — 동명이인 빈도 확인 후 결정
- 카카오 등 외부 알림 채널
- report.html 기능 변경 — 단체 대회 내용 표시하지 않음 (legacy 방향)

## 4. 등장인물 (Actor)

| Actor | 페이지 | 권한 |
|---|---|---|
| 오너 (나) | ops.html | 기록 소스 매핑, 시스템 관리 (`DMC_OWNER_PW`) |
| 운영자 | group.html, report.html | 참가자 등록, 기록 확정 (운영자 비밀번호) |
| 개인 회원 | my.html | 본인 기록 확인·확정 |

## 5. 유저 시나리오

### 시나리오 A: 운영자가 단체 대회 등록 + 참가자 선택 (D-30)

1. group.html 접속 → "고러닝 예정 대회" 목록
2. "2026 서울국제마라톤" → **단체 대회로 등록** 클릭
3. 정회원 명단 팝업 → 닉네임 검색·체크박스 선택 → 저장
4. 이후 참가자 목록은 대회 전 언제든 수정 가능

### 시나리오 B: 오너가 기록 소스 매핑 + 수동 스크랩 (D-7 ~ D+0)

1. ops.html 접속 (오너 비밀번호 입력)
2. 단체 대회 목록 → 기록 소스 입력 (예: `smartchip_202650000099`)
3. 저장 → 즉시 스크랩 트리거 → group.html에 "✅ 스크랩 완료" 상태 표시
4. **소스 미매핑 시 정책**: 자동 스크랩 없음. 오너가 ops.html에서 수동 입력 후 즉시 트리거

### 시나리오 C: 대회 당일 자동 스크랩 (D+0, 15:00 KST)

1. Cloud Scheduler 실행 → 오늘 단체 대회 중 **아직 스크랩되지 않은** 대회만 실행
   - 오너가 이미 수동 트리거한 대회는 건너뜀
2. 소스 미매핑 대회 → group.html 경고 배너 + ops.html 링크

**오너 알림 (별도 구현):**
- D-1 및 D-0 오전: "소스 미매핑 단체 대회 N건" 알림을 ops.html 접속 시 표시

### 시나리오 D: 운영자가 갭 탐지 결과 확인·기록 확정 (D+0 이후)

1. group.html → 해당 대회 카드 → 결과 확인
2. ✅ 자동 매칭: 기록·순위·배번 확인 후 일괄 또는 개별 확정
3. 🟡 동명이인: 후보 기록 2개 중 해당 회원 선택 후 확정
4. 🔴 기록 없음: 재스크랩 / DNS / DNF 중 선택 → 처리하면 완료

> 모든 확정 액션은 기존 `confirm` API를 호출한다.

## 6. 기능 요구사항

### 6-1. group.html

| ID | 요구사항 |
|---|---|
| G-01 | 고러닝 연동 예정 대회 목록 표시 |
| G-02 | 대회를 단체 대회로 등록/해제 |
| G-03 | 정회원 명단에서 체크박스로 참가자 선택 (닉네임 정렬, 검색 필터, hidden 제외) |
| G-04 | 참가자 목록 대회 전 언제든 수정 가능 |
| G-05 | 기록 소스 상태 표시 (읽기 전용): ⚪ 미지정 / ✅ 확정, ops.html 링크 제공 |
| G-06 | 소스 미매핑 + 대회 당일 → 경고 배너 |
| G-07 | 스크랩 완료 후 참가자별 기록 표시: 닉네임, 기록, 순위, 배번 |
| G-08 | ✅ 자동 매칭 일괄 확정 버튼 (기존 `confirm` API 호출) |
| G-09 | 🟡 동명이인 → 후보 기록 최대 3개 표시, 초과분은 접어서 표시 → 선택 후 확정 (기존 `confirm` API 호출) |
| G-10 | 🔴 기록 없음 → 재스크랩 / DNS / DNF 처리 |

### 6-2. ops.html 변경

| ID | 요구사항 |
|---|---|
| O-01 | 오너 전용 비밀번호 분리 (`DMC_OWNER_PW`), 서버 검증. 오너 전용 API는 매 호출마다 `ownerPw`를 request body에 포함 |
| O-02 | 단체 대회 목록 + 기록 소스 입력 UI |
| O-03 | 소스 저장 시 즉시 스크랩 트리거 (시간 무관) |
| O-04 | 접속 시 소스 미매핑 단체 대회 N건 경고 표시 (D-1, D-0) |

### 6-3. 백엔드 API

| ID | action | 설명 | 권한 |
|---|---|---|---|
| A-01 | `group-events` GET | 단체 대회 목록 조회 | 운영자 |
| A-02 | `group-events` promote | 고러닝 이벤트 → 단체 대회 승격 | 운영자 |
| A-03 | `group-events` participants | 참가자 저장 | 운영자 |
| A-04 | `group-events` source | 기록 소스 매핑 | **오너 전용** |
| A-05 | `group-events` scrape | 수동 스크랩 트리거 | **오너 전용** |
| A-06 | `group-events` gap | 갭 탐지 결과 조회 | 운영자 |
| A-07 | `verify-admin` | `owner` / `operator` role 반환 | — |
| A-08 | 기존 `confirm` | 기록 확정 (재사용, 변경 없음) | 운영자 |

### 6-4. Cloud Scheduler

| ID | 요구사항 |
|---|---|
| S-01 | 매일 15:00 KST 실행 |
| S-02 | 오늘 날짜(KST) = 대회일이고, 소스 매핑됐으며, **아직 스크랩되지 않은** 단체 대회만 실행 |
| S-03 | 스크랩 실패 시 group.html 오류 상태 표시 |

## 7. 데이터 스키마 변경

> 상세 필드 설계는 Tech Spec에서 결정

`race_events`에 단체 대회 관련 필드 추가:
- 단체 대회 여부 플래그
- 참가자 목록 (memberId, realName)
- 기록 소스 정보
- 자동 스크랩 상태

## 8. 정책 정의

| 항목 | 정책 |
|---|---|
| 단체 대회 생성 방식 | 고러닝 등재 대회에서 운영자가 승격 |
| 참가자 수정 가능 기간 | 대회일 이전 언제든지 |
| 스크랩 트리거 — 수동 | 오너가 ops.html에서 소스 저장 시 즉시 실행 (시간 무관) |
| 스크랩 트리거 — 자동 | 대회일 오후 3시 KST, 아직 스크랩되지 않은 대회만 |
| 소스 미매핑 시 | 자동·수동 스크랩 모두 불가. 오너가 ops.html에서 소스 입력 후 즉시 트리거 |
| 갭 탐지 매칭 키 | memberId 기준 |
| 동명이인 처리 | 갭 확인 시점에 운영자가 후보 최대 3개 중 선택 |
| DNS/DNF 처리 | 갭 확인 시점에 운영자가 선택 (재스크랩 / DNS / DNF) |
| DNS/DNF 저장 | race_results에 저장. 노출 여부는 추후 결정 |
| 날짜 기준 | 전체 KST |
| 인증 방식 | 오너 전용 API는 매 호출마다 ownerPw를 request body에 포함 |
| report.html | 단체 대회 내용 표시 안 함 (legacy 방향) |
