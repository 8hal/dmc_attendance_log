## 배번 입력 셀프서비스 UI 개발 (v0.12.1)

### 배경
단체 대회에서 동명이인 이슈 해결을 위해 회원이 직접 배번을 입력할 수 있는 페이지 개발.
기존에는 운영진이 대회 당일 수동 매칭하느라 30분 이상 소요되었음.

### 구현 내용

#### 1. Backend API (`functions/index.js`)
- **새 API**: `POST /race?action=group-events&subAction=update-bib`
- **파라미터**: `eventId`, `nickname`, `bib`
- **검증**:
  - 필수 파라미터 체크
  - 배번 형식 검증 (빈 문자열 방지)
  - 참가자 확인
  - **배번 중복 검증** (서버 레벨)
- **응답**: `{ ok: true, message: "배번이 저장되었습니다" }`

#### 2. Frontend (`my-bib.html`)

**주요 기능:**
- ✅ 참가자 목록 UI (닉네임 입력 방식 → 목록 선택 방식)
- ✅ 실시간 검색 기능 (닉네임 필터링)
- ✅ 배번 입력 상태 표시 (입력완료/미입력)
- ✅ 입력된 배번 표시 (#12345)
- ✅ 배번 중복 검증 (클라이언트 레벨)
- ✅ 저장 후 토스트 메시지 + 자동 목록 복귀
- ✅ "목록으로 가기" 뒤로가기 버튼
- ✅ 로컬 스토리지 연동
  - 출석 페이지 키 재사용: `marathon_att_nickname`
  - 출석 체크한 회원은 배번 페이지 접속 시 바로 입력 화면 진입
- ✅ 환경 자동 감지 (localhost=에뮬, 실서버=프로덕션)
- ✅ `distance` 없을 때 처리 (undefined → `-`)

**UI/UX 개선:**
- 대회명 폰트 크기 증가 (14px → 16px, semibold)
- 검색창 추가
- 종목 정보 표시

#### 3. 데이터 업데이트 스크립트
- **파일**: `scripts/update-participants-distance.js`
- **목적**: 엑셀 파일(`2026 경기 마라톤 참가자 명단.xlsx`)에서 distance 정보 추출

- **결과**: 85명 전원 distance 업데이트 완료 (HALF, FULL, 10K)

## 동명이인 코스 매칭 필터링 및 distance 표시 통일 (v0.13.0)

### 배경
단체 대회 스크래핑 시 동명이인 케이스(3~5%)가 발생하여 수동 확인이 필요했음.
배번 입력 기능(v0.12.1)으로 종목 정보가 확보되면서, 종목 기반 자동 필터링이 가능해짐.

### 구현 내용

#### 1. Backend - Distance 매칭 필터링 (`functions/lib/scraper.js`)

**로직:**
```javascript
// 참가자 종목 정규화
const participantDistance = normalizeRaceDistance(m.distance);

// 검색 결과 필터링
const matched = found.filter(r => {
  const resultDistance = normalizeRaceDistance(r.distance);
  return resultDistance === participantDistance;
});

// Fallback: 매칭 실패 시 원본 유지
if (matched.length > 0) {
  filteredResults = matched;
} else {
  console.warn(`distance 매칭 실패, 원본 유지`);
}
```

**효과:**
- 동명이인 + 다른 종목 → 자동으로 1개 선택 (ambiguous 해소)
- 동명이인 + 같은 종목 → ambiguous 유지 (수동 선택 필요)
- distance 정보 없음 → 원본 유지 (기존 동작 보존)

#### 2. Distance 정규화 별칭 확장 (`functions/lib/raceDistance.js`)

**추가된 별칭:**
- `"하프 마라톤"` → `"half"`
- `"풀 마라톤"` → `"full"`

**이유:**
실제 스크래핑 결과에서 띄어쓰기 포함 표현이 자주 나타남.
정규화 실패 시 필터링이 작동하지 않음.

#### 3. Frontend - Distance 표시 통일

**공통 유틸리티 생성:**
- 파일: `assets/distance-utils.js`
- 기능: `formatDistance()`, `distBadge()`, `escapeHtml()`
- 기준: `my.html`의 기존 로직 채택

**적용 페이지:**
- `my-bib.html`: 기존 `formatDistance` 제거 → 공통 유틸 사용
- `group-detail.html`: 기존 `distanceLabel` 제거 → 공통 유틸 사용
- `my.html`: 이미 동일 로직 사용 (변경 없음)

**일관성:**
- "하프" 대신 "하프"로 통일
- "풀" 대신 "풀"로 통일
- "10K" → "10K" (그대로)
- null/undefined → `-` (대시)

#### 4. 단위 테스트 (`scripts/test-distance-matching.js`)

**8개 테스트 케이스:**
1. 동명이인 + 다른 종목 → 1개 필터링 ✅
2. 동명이인 + 같은 종목 → ambiguous 유지 ✅
3. distance 정보 없음 → 원본 유지 ✅
4. 정규화 테스트 (HALF/half/하프 → half) ✅
5. 참가자 종목 불일치 → 원본 유지 ✅
6. 단일 결과 + 종목 일치 ✅
7. 3명 동명이인 + 2명 같은 종목 → 2명 필터링 ✅
8. unknown distance → 원본 유지 ✅
