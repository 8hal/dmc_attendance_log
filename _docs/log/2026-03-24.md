## "이 대회 뛰셨나요?" 프로액티브 제안 기능 설계

주말 대회 후 회원이 my.html 접속 시, search_cache 기반으로 자동 제안.
- search_cache found:true 미확정: **15,736건** (152명)
- 검색 퍼널 완전 생략, 원탭 확정
- 동명이인도 "이 대회 뛴 거 맞아?" 맥락에서 대폭 경감
- 주간 스크래퍼(토/일 15:00) → 월요일 접속 시 배너 제안

### 발견: 주간 스크래퍼 ↔ search_cache 미연결

코드 리뷰에서 발견 — **주간 스크래퍼(weeklyDiscoverAndScrape)는 search_cache에 쓰지 않음.**
- 주간 스크래퍼 → `scrape_jobs.results[]`에만 저장 (대회 단위, 전 회원 혼합)
- 개인 검색 / 프리워밍 → `search_cache`에 저장 (회원×대회 단위)
- 두 시스템은 개발 시점이 달라 완전히 분리되어 있음
- **해결:** `scrapeEvent()`에서 search_cache 동시 쓰기 추가 (TASK_01 Step 0)
