# 2026-04-20 작업 일지

## 배번 양도 기능 버그 수정 (v0.14.1)

### 배경
4월 18-19일에 구현된 배번 양도 기능에서 연속된 양도 시도 시 잘못된 회원이 선택되는 버그 발견.
구체적으로 "다니엘 → 마운틴" 양도 후 "바람 → 데이" 양도 시도 시 "바람 → 마운틴"으로 표시됨.

### 문제 원인
- `renderTransferList` 함수에서 `member.memberId`를 사용했지만, `action=members` API는 `member.id`로 반환
- 결과: `memberId`가 빈 문자열(`""`)로 설정되어 `findIndex`가 항상 첫 번째 빈 항목(이전 양도로 추가된 비참가자) 반환

### 해결 방법
```javascript
// renderTransferList 함수 수정
const memberId = member.id || member.memberId;  // API 응답의 'id' 우선 사용
```

### 배포
- 디버그 로그 제거 완료
- 코드 정상 상태 복원

## 그룹 대회 races 페이지 대회명 누락 버그 수정 (v0.14.2)

### 배경
그룹 대회 확정 후 `races.html`에서 대회명이 표시되지 않는 문제 발견.

### 문제 원인
- `promote` API에서 그룹 대회 생성 시 `eventName`만 설정하고 `primaryName` 미설정
- `confirmed-races` API는 `primaryName`을 우선 사용
- 결과: `primaryName`이 없는 그룹 대회는 대회명 공백

### 해결 방법

**1. Backend 수정 (`functions/index.js` 라인 2716)**
```javascript
await ref.set({
  eventName,
  primaryName: eventName,  // ← 추가
  eventDate,
  isGroupEvent: true,
  // ...
}, { merge: true });
```

**2. 기존 데이터 마이그레이션**
- 스크립트: `scripts/fix-group-event-primary-names.js`
- 실행 결과: 1개 그룹 대회 수정 (제24회 경기마라톤대회)

### 배포
- Pre-deploy test: 통과 42 / 실패 0
- Functions 배포 완료
- 데이터 마이그레이션 완료

## 그룹 대회 재확정 기능 테크 스팩 작성 (기획)

### 배경
그룹 대회 확정 후 잘못된 기록(DNS/DNF 누락, 코스 오류)을 수정할 방법이 필요함.

### 설계 과정

**1단계: 초안 설계 (서브 에이전트)**
- 옵션 4개 비교 분석
- 추천: 옵션 1 + 옵션 2 하이브리드 (재검토 모드 + 코스 변경)

**2단계: 코스 수정 기능 추가 요구**
- "⋮" 메뉴에 "코스 변경" 추가
- 지연된 일괄 저장 방식
- docId 변경 처리 (코스 변경 시)

**3단계: 개인 확정 기록 충돌 검토**
- 질문: "race_result를 불러와서 고치는 거면 개인 확정 등과 부딪힐 가능성은 없나?"
- 분석: 5가지 옵션 검토 (Soft Lock, Merge, 별도 컬렉션 등)
- 결정: **단순 덮어쓰기** (Last Write Wins)

**4단계: 정책 단순화**
- "단체 대회는 개인이 입력할 가능성이 낮음"
- **정책**: 그룹 대회는 운영진, 개인 대회는 개인
- **구현**: `confirmSource: "operator"`만 삭제, `"personal"` 보존

**5단계: 팀장 리뷰**
- Critical Issue 3개 발견:
  - C1: 개인 확정 기록 삭제 위험 (P0, 배포 차단)
  - C2: replaceMode 검증 누락
  - C3: 동시성 문제 (Race Condition)
- Major Issue 5개 발견:
  - M1: 스팩 오류 (bulk-confirm 이미 구현됨)
  - M5: 네트워크 오류 시 부분 삭제 위험
- 배포 시간 재산정: 3-4일 → 0.5-1일 (5시간)

**6단계: 스팩 수정**
- `confirmSource` 필터 추가 (P0)
- `replaceMode` 파라미터 제거 (불필요)
- 배번 양도 시나리오 제거 (별도 스팩 분리)
- 테스트 케이스 추가 (TC 7.5, 7.6: 개인 확정 보호)

**7단계: 원칙 일관성 검토**
- 핵심 원칙 명확화 (`confirmSource` 역할 추가)
- Mock UI 경고 메시지 수정
- API 명세 주석 보강
- 코스 변경 시 docId 변경 로직 주석 추가

### 최종 스팩

## 메모

### 배번 양도 기능 디버깅 교훈
- **Debug Mode 활용**: 광범위한 instrumentation으로 근본 원인 정확히 파악
- **API 응답 검증**: `member.id` vs `member.memberId` 불일치 발견
- **런타임 증거**: 로그 없이는 추측만 했을 버그를 정확히 수정

### 스팩 작성 교훈
- **기존 코드 확인 필수**: `bulk-confirm` API 이미 구현되어 있었음
- **팀장 리뷰 가치**: Critical Issue 3개 사전 발견
- **원칙 일관성**: 문서 전체가 핵심 원칙과 일치해야 함
- **단순화의 중요성**: 복잡한 Merge 로직보다 단순 덮어쓰기가 관리 용이

### 정책 결정
- **그룹 대회**: 운영진이 관리 (`confirmSource: "operator"`)
- **개인 대회**: 개인이 입력 (`confirmSource: "personal"`)
- **충돌 해결**: 개인 확정 기록 보호 (운영자 재확정 시 개인 기록은 삭제 안 함)
