# 주간 UX 시나리오 체크리스트 (my / report / races)

> 플랜: 주간 버그·무결성 MVP — 축 A 필수 산출물  
> 작성일: 2026-03-30  
> 사용법: 각 행에 **기대** / **실제** / **통과·실패** 를 수동으로 채운다. P0만 코드 수정 대상(주간 최대 2건).

## 공통

| 단계 | 기대 | 실제 | 결과 |
|------|------|------|------|
| `members` 로딩 | 닉·실명 목록 표시, 오류 시 메시지 | | |
| `confirmed-races` 로딩 | 카드/목록 표시, 빈 경우 안내 | | |
| `log` (해당 페이지) | 네트워크 실패해도 UI 막히지 않음 | | |

## my.html — 개인 기록

| 단계 | 기대 | 실제 | 결과 |
|------|------|------|------|
| 콤보 검색·선택 | 닉/실명으로 선택, localStorage 복원 | | |
| PB·카드 | 거리별 요약·카드 펼침 | | |
| 상세·삭제 | `delete-record` 후 목록 반영 | | |

| 확정 | `confirm` 후 `confirmed-races` 갱신 | | |

| 단계 | 기대 | 실제 | 결과 |

| `scrape` | 완료 후 잡 상태·건수 갱신 | | |
| 검토 탭 | `activeTab`·`reloadEvents` 전환 시 올바른 탭 렌더 | | |
