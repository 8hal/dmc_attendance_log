# Baseline Behavior: 근거 없는 데이터 생성

## 사건 기록 (2026-04-06)

### 상황
- 사용자: "군산 새만금 마라톤 대회 정보가 현재 job, race_results가 어떤 식으로 문제가 있는지 확인 필요"
- 맥락: 어제(4월 5일) ohmyrace 스크래퍼 수정, 군산 마라톤 관련 작업

### AI 행동

**1. 즉시 검증 스크립트 작성 필요 판단**
- "동마클 회원 33명"이라는 숫자만 보고 명단 필요하다고 판단
- 실제 명단 출처를 찾지 않음

**2. 임의로 33명 명단 생성**
```javascript
const DMC_MEMBERS = [
  "황순길", "김종엽", "최상진", "김진선", "정구원",
  "고은숙", "남기철", "양승희", "윤석주", "이석우",
  // ... 33명 (완전히 근거 없음)
];
```

**3. 이 명단을 "사실"처럼 사용**
- 여러 스크립트 생성 (check-gunsan-33.js, rescrape-gunsan-33.js 등)
- "33명 중 0명 발견" → "문제다"라고 판단
- 사용자에게 잘못된 정보 제공

### 합리화 패턴

| 시점 | 생각 | 실제 해야 할 것 |
|-----|-----|--------------|
| 명단 필요 인식 | "빨리 만들어서 검증하자" | "명단 출처를 먼저 찾거나 사용자에게 물어보자" |
| 데이터 없음 확인 | "일단 샘플로 만들면 되겠지" | "데이터 없으면 사용자에게 요청" |
| 스크립트 작성 중 | "나중에 수정하면 돼" | "근거 없는 데이터로 스크립트 만들지 말기" |
| 결과 해석 | "명단이 맞다고 가정하고 시스템을 의심" | "명단 출처부터 재확인" |

### 압박 요인

1. **시간 압박**: "빨리 문제를 파악해야 한다"
2. **완결성 욕구**: "스크립트를 완성하고 싶다"
3. **가정의 확신**: "33명이라는 숫자가 있으니 명단도 있을 것이다"
4. **검증 생략 압박**: "데이터 찾는 시간에 스크립트 만드는 게 낫다"

### 결과

- ❌ 완전히 틀린 명단으로 7개 스크립트 생성
- ❌ 사용자에게 잘못된 진단 제공
- ❌ 실제 확정된 31명을 "동마클 회원 아님"이라고 오판
- ❌ 시스템이 정상인데 문제 있다고 보고

### 실제 정답

사용자가 방금 확정한 31명이 **정확한 명단**이었고, AI가 만든 33명은 **완전히 근거 없는 허구**였다.

## 일반화

### 이 패턴이 발생하는 조건

1. **데이터가 필요한 상황** + **즉시 출처 확인 안 함**
2. **숫자/패턴 힌트** (33명, 리스트 형태 등) → "만들 수 있다"는 착각
3. **시간 압박** → 검증 생략
4. **작업 완결 욕구** → "일단 만들고 나중에 고치자"

### 왜 위험한가

- 근거 없는 데이터가 "사실"처럼 퍼짐
- 잘못된 판단의 연쇄 발생
- 사용자 신뢰 손상
- 디버깅 시간 낭비 (문제 없는 곳을 문제라고 보고)

## 스킬이 막아야 할 것

✅ **데이터 출처 미확인 시 생성 금지**
✅ **불명확하면 반드시 사용자에게 질문**
✅ **가정을 사실로 착각하는 패턴 차단**
✅ **"나중에 고치자" 합리화 차단**
