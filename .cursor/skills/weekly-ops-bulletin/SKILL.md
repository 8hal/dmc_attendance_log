# 주간 운영진 안내 (Weekly ops bulletin)

## 언제 쓰는가

- **주간 회고가 끝난 뒤**, 카톡·이메일·노션 등으로 **운영진에게 한 주 요약**을 보낼 때.
- 수신자는 **IT 배경이 거의 없음**. 짧은 문장, 불릿, 전문 용어 금지.

## 절차

1. **근거 수집:** `_docs/log/YYYY-MM-DD.md` 해당 주 분(월~일)과 필요 시 `_docs/knowledge`·최근 커밋 메시지를 읽는다. 대화만의 기억으로 쓰지 않는다.
2. **사실 검증:** 코드·일지에 없는 성과(예: “시트 전부 입력 완료”)는 **확인되지 않은 채로 단정하지 않는다.** 불확실하면 부드럽게 (“반영을 진행했다”) 또는 일지에 한 줄 남긴 뒤 문구에 쓰라고 안내한다.
3. **기술 표현 교정:**
   - “우회 로직” 대신 **“사이트 제한에 맞춰 조회하는 방식”** 등 일상어.
   - 리포트 스크랩: **“회원을 고른 뒤에만 가져온다. 전체 선택 가능”** — **“대회 참가자만 자동 조회”**는 코드와 다르면 쓰지 않는다 (`report.html` 회원 선택 모달, `memberRealNames`).
4. **구성 (2~3블록):**
   - **지난 주:** 운영진이 알면 좋은 일 2~4불릿.
   - **이번 주:** 뭘 할지 한 블록 — **누가 QA/점검하는지** 사용자 지시에 따른다.
   - **부탁 (선택):** 불편 시 연락 달라 — 안 넣어도 됨.
5. **카톡 등 마크다운 없는 채널:** `■` 섹션 + `·` 불릿, 블록 사이 빈 줄. 아래 **정식 예시** 참고.
6. **길이:** 본문 **15줄 이내** 권장. 제목 한 줄 + 불릿.
7. **수치:** `N건`, `N명` 등은 `.cursor/rules/data-verification-gate.mdc` — **일지·API 출력에 없으면 안 쓰거나** “일지 기준”을 붙인다.

## 카톡용 정식 예시 (마크다운 없음 · 사용자 확정본)

아래는 그대로 붙여 넣기 좋은 형식. 내용만 주차에 맞게 바꾼다.

```
동마클 기록 시스템 주간 공유

■ 지난 주에 한 일

· …

■ 이번 주에 할 일

· …
```

(선택) 마지막에 한 블록 추가:

```
■ 부탁

· 쓰시다가 불편하거나 이상한 점이 있으면 언제든지 알려 주세요. 반영할 때 도움이 됩니다.
```

## 출력 형식 예시 (노션·이메일 등 마크다운 가능할 때)

```markdown
**동마클 기록 — 운영진께**

**지난 주에 한 일**
- …

**이번 주에 할 일**
- …

**부탁 한 가지** (선택)
- …
```

## 관련 룰·스킬

- `.cursor/rules/weekly-ops-bulletin.mdc` — 트리거·금지 패턴 요약
- `verify-numbers` — 수치 포함 시
- `end-of-day` / 회고 로그 — 입력 소스가 `_docs/log`인지 확인

## 금지

- 배포 URL, 함수 이름, 컬렉션명을 운영진 문구에 노출하지 않는다.
- 검증 없는 구체 숫자 단정.
- 운영진에게 “직접 QA해 달라”고 쓰는데, 실제로는 담당자만 QA하는 경우 — **항상 사용자가 정한 역할과 맞출 것.**
